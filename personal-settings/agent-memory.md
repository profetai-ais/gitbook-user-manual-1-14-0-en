# Personalization

## Introduction

Personalization is a feature that allows Agents to "remember" and "continue using" key information in subsequent interactions, improving conversation continuity and task processing efficiency. When users provide preferences, background information, common settings, project context, or fixed formats during interactions, the system can organize this content into reusable memories, allowing Agents to automatically reference them in future conversations, reducing repeated explanations and back-and-forth confirmation.

For example, a user's preferred name, tone style, output format (bullet points/table/paragraphs), frequently handled product or project names, workflow preferences, common decision principles, and standards that need to be consistently maintained over time can all be stored as memories and applied at appropriate moments.

The memory feature is accessible through the "Personalization" tab in the profile icon at the top-right corner.

### Memory

Memory is divided into "Save Memory" and "Reference Memory."

<figure><img src="../.gitbook/assets/image (128).png" alt=""><figcaption></figcaption></figure>

* **Save Memory**: When enabled, allows the Agent to save memories.
* **Reference Memory**: When enabled, allows the Agent to reference memories from the memory store when responding.

## Manage Memory

The entry point for memory management is the "Manage" button. Memory management is divided into "Personal Memory" and "Agent Memory."

<figure><img src="../.gitbook/assets/image (129).png" alt=""><figcaption></figcaption></figure>

### Personal Memory

When Reference Memory is enabled, memories stored here can be accessed by all Agents. Personal memory can store a maximum of 50 entries.

> Note: When the storage limit is exceeded, the system will automatically delete the least frequently used memory entries.

<figure><img src="../.gitbook/assets/image (130).png" alt=""><figcaption></figcaption></figure>

### Agent Memory

When Reference Memory is enabled, an Agent can access memories from its own memory store. Each Agent can store a maximum of 50 memory entries.

> Note: When the storage limit is exceeded, the system will automatically delete the least frequently used memory entries.

## How to Save Memories

<figure><img src="../.gitbook/assets/image (133).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (134).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (135).png" alt=""><figcaption></figcaption></figure>

1. Make sure "Save Memory" is enabled before using the memory feature.
2. In the conversation interface, select the Agent you want to chat with and enter the content you want to be remembered.
3. After the memory is saved successfully, the Agent will prompt you that you can go to "Personalization → Memory Management" to view the memory.

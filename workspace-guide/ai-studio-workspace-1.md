# Conversation Space

## Page Navigation

The interface is primarily composed of two sections:

* **Conversation History**: Users can start new conversations or quickly access previous conversation records.
* **Chat Area**: The place where users converse with an Agent.

> Note: The following screenshots are for reference only. Actual conversation history, Agents, and prompt templates may vary depending on your environment.

The Conversation History area is divided into two sections:

* **Recent Conversations**: Generated conversation threads are stored here. Click any thread to open it in the Chat Area.
* **Recent Agents**: Agents you have previously conversed with are listed here.

## **Chat Session Files**

During a session with an Agent, two types of files may be involved. Click the file icon within "…" to **view** them, and perform **download** or **delete** operations based on your permissions:

### File Icons

* **Reference Files**: Files uploaded by the user as prompt attachments.
* **Work Files**: Files generated by the Agent in its responses or outputs.

Download and access permission notes:

* The system determines whether downloads are permitted based on the knowledge base / dataset download access control and the user's login status.
* If the user is not authorized or has not completed verification, the system may block the download, hide the download entry, or display an insufficient permissions message.
* "Work Files" generated by the Agent in its responses are subject to the same access rules if their source involves controlled data.

## **Usage Quota**

The **Usage Quota** area lets users clearly understand their Token and cost consumption when using AI models in the **AI Studio Workspace**. Tokens represent the smallest unit of data processed, including user input and model output. This feature helps users track and manage their usage in real time.

> Note: By default, quota usage resets automatically each day.

## Provide Feedback on Agent Response Quality

Users can provide feedback on individual Agent responses to improve memory retrieval quality and Agent response accuracy. Feedback buttons are displayed below each Agent response, and users can choose to provide positive or negative feedback.

### Provide Positive Feedback

If the Agent's response meets the user's needs, they can click the "Like" button below the response to open the positive feedback window.

<figure><img src="../.gitbook/assets/image (334).png" alt=""><figcaption></figcaption></figure>

In the positive feedback window, users can select a reason for the feedback, such as:

* Successfully resolved my task
* Followed the instructions
* Good response quality
* Fast and efficient
* Autonomous actions were helpful
* Other reasons

Users can also add more details in the additional comments field, such as which parts of the response were particularly helpful or met expectations. This field is optional.

### Provide Negative Feedback

If the Agent's response does not meet the user's needs, they can click the "Dislike" button below the response to open the negative feedback window.

<figure><img src="../.gitbook/assets/image (335).png" alt=""><figcaption></figcaption></figure>

In the negative feedback window, users can select a reason for the feedback, such as:

* Misunderstood my intent
* Content is incorrect
* Response is incomplete
* Did not follow the instructions
* Content is unsafe or inappropriate
* Other reasons

Users can also add more details in the additional comments field, such as what was incorrect in the response, what information was missing, or how they would like the Agent to improve. This field is optional.

### View Submitted Feedback

After submitting feedback, the corresponding feedback button will show a selected state. Users can click the feedback button again to open the action menu.

Click "View Feedback" to see the previously submitted feedback, including the selected reason and any additional comments.

### Withdraw Feedback

If a user wants to cancel submitted feedback, they can click the selected feedback button and choose "Remove Feedback" from the action menu.

After removal, the response returns to an unrated state, and the user can re-select positive or negative feedback.

> Note: Each Agent response can receive one instance of feedback. If feedback has already been submitted, users can view or remove it before resubmitting new feedback. Feedback is used to improve future Agent response quality; users are encouraged to clearly describe in the additional comments why a response met or did not meet their expectations.

## Canvas

The Canvas feature helps users display AI-generated content in the right-side workspace for easier review, editing, and organization. When users converse with the AI and generate lengthy or complex content, Canvas presents it in a clearer layout, such as documents, tables, charts, code, or other visual content.

Canvas is suited for scenarios that require extended viewing and adjustment of content, such as writing documents, organizing reports, analyzing data, reviewing charts, or continuously modifying the same content during a conversation. Users can discuss with the AI on the left while viewing or maintaining Canvas content on the right, reducing the need to repeatedly search through chat messages.

### Invoke Canvas

<figure><img src="../.gitbook/assets/image (106).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (108).png" alt=""><figcaption></figcaption></figure>

1. Click the "+" icon in the lower-left corner, select Canvas, and the feature will appear in the input box.
2. Enter your request in the chat window and ask the AI to generate a document, table, chart, or other content.
3. When the system generates content that can be displayed in Canvas, a Canvas card will appear on screen.
4. Click the Canvas card, and the system will open the content in the right-side Canvas workspace, where users can view the full content.
5. If the content needs adjustment, continue requesting changes from the AI through the chat window, or directly edit the canvas content if the system supports it. Modified content will be displayed in Canvas for users to continue reviewing and organizing.

> Note: Canvas is best suited for content that is lengthy, complex, or requires repeated viewing and modification. For short text responses, the system may display them directly in the chat window without opening Canvas.\
> The supported operations may differ depending on content type; for example, documents, tables, charts, and code may be presented and edited differently. If Canvas content is AI-generated, users are advised to verify the content before using it, and may request further corrections or additions from the AI if needed.



## Speech to Text

The voice dictation feature helps users input text through speech. Users simply enable the microphone and start speaking; the system will convert the speech to text, reducing manual input time and improving efficiency for filling in data or entering text.

<figure><img src="../.gitbook/assets/image (142).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (143).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (144).png" alt=""><figcaption></figcaption></figure>

1. Go to the Agent conversation page and click the microphone icon next to the text input box.
2. If the system prompts for microphone permission, allow the system to use the microphone.
3. Clearly speak the content you want to input.
4. When you are done recording, click the "Confirm" button.
5. Once transcription is complete, the text will appear in the input field. Verify that the text is correct; if there are typos or incomplete meaning, you can manually edit it before sending.

> Note: To improve speech recognition accuracy, use this feature in a quiet environment and speak clearly at a steady pace. Excessive background noise, speaking too fast, or unclear pronunciation may affect recognition results.\
> Speech transcription requires some processing time; please wait for the system to complete transcription after clicking the Confirm button. The transcribed text may contain typos, omissions, or incomplete punctuation. Be sure to review the content before sending.
